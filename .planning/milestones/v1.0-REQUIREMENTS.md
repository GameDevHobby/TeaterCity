## Requirements

### Room Selection & Menu

- [x] SEL-01: Player can tap a room to select it
- [x] SEL-02: Selected room shows visual highlight
- [x] SEL-03: Room menu appears with options: Edit Furniture, Edit Room, [Room-type placeholder]
- [x] SEL-04: Room-type-specific placeholder button for future features (e.g., theater scheduling)

### Room Editing

- [x] EDIT-01: Player can resize room by adjusting walls
- [x] EDIT-02: Resize blocked if furniture would be invalidated
- [x] EDIT-03: Doors reset on resize, player re-places them
- [x] EDIT-04: Player can delete a room
- [x] EDIT-05: Walls heal (fill in) at door locations when room deleted

### Door Editing

- [x] DOOR-01: Player can add doors to room walls
- [x] DOOR-02: Player can remove doors from room
- [x] DOOR-03: Doors can only open to empty space (not into adjacent rooms)

### Furniture Management

- [x] FUR-01: Player can select furniture by tapping directly in room
- [x] FUR-02: Player can select furniture from list panel
- [x] FUR-03: Player can move furniture to new position
- [x] FUR-04: Player can add new furniture (within room type constraints)
- [x] FUR-05: Player can delete furniture (respecting room type minimum requirements)

### Overlap Prevention

- [x] OVR-01: Adjacent rooms can share wall boundaries
- [x] OVR-02: Room resize cannot encroach on furniture from other rooms
- [x] OVR-03: Validation prevents invalid room/furniture configurations

### Persistence

- [x] PER-01: Rooms auto-save on changes
- [x] PER-02: Furniture placements auto-save on changes
- [x] PER-03: Room settings persist through game quit/restart
- [x] PER-04: Data stored in user:// JSON file
- [x] PER-05: Admin menu for save management (reset) -- feature-flagged

### Testing

- [x] TEST-01: Unit tests for room editing operations
- [x] TEST-02: Unit tests for furniture management operations
- [x] TEST-03: Unit tests for persistence layer
- [x] TEST-04: Integration tests for edit workflows
